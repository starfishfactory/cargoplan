# 카고플랜 자체 구축 가능 기능 정밀 분석

## 분석 목적

서프컴퍼니(surff.kr)가 제공하는 기능들을 분석하여, 카고플랜이 **서프와 계약하지 않고 직접 구축할 경우** 유효한 기능이 무엇인지 파악한다.

## 자체 구축 유효성 분석 (5단계 평가)

### 평가 기준
- **구축 난이도**: 기술적 복잡도 (낮음/중간/높음)
- **데이터 확보**: 필요 데이터의 자체 확보 가능성
- **카고플랜 적합성**: 포워더 비즈니스에 실질적 도움 여부
- **종합 판정**: ✅ 구축 추천 / ⚠️ 조건부 추천 / ❌ 비추천

---

### ✅ Tier 1: 즉시 구축 추천 (높은 실효성)

#### 1. 카고 트래킹 (BL 기반 화물 추적)
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **중간** |
| 데이터 확보 | 선사 API 연동 필요 (공개 API 다수 존재) |
| 카고플랜 적합성 | **매우 높음** — 기존 고객(KT&G, 명보 등) 서비스 강화 핵심 |
| 핵심 기능 | BL 등록, 선사별 추적, 일괄 등록/내보내기, ETD/ETA 상태 |
| 자체 구축 이점 | 고객 데이터 100% 자사 보유, 커스터마이징 자유 |

**근거**: 트레드링스 ShipGo Plug-in이 무료로 제공할 정도로 핵심 니즈. 포워더가 직접 운영하면 고객 신뢰도 상승.

#### 2. 선박 입출항 알림
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **중간** |
| 데이터 확보 | 항만공사/해양수산부 공공 API 활용 가능 |
| 카고플랜 적합성 | **높음** — 실무 업무 효율화 직결 |
| 핵심 기능 | 터미널 검색, MY VESSEL 등록(최대 10척), 일정 변경 알림 |
| 자체 구축 이점 | 내부 운영팀 업무 도구로도 활용 가능 |

**근거**: 반입마감시간, 접안/출항 예정일시 등 실무에 즉시 활용 가능한 데이터.

#### 3. Booking 관리 시스템
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **중간~높음** |
| 데이터 확보 | 자사 부킹 데이터 (이미 보유) |
| 카고플랜 적합성 | **매우 높음** — 핵심 업무 디지털화 |
| 핵심 기능 | FCL 정보 관리, 부킹 상태 추적, 고객별 관리 |
| 자체 구축 이점 | 기존 업무 프로세스 그대로 반영 가능 |

**근거**: 서프에서 사이드바로 제공하는 수준이지만, 포워더 입장에서는 내부 운영 시스템으로 확장 가능.

#### 4. 실시간 알림 시스템 (Notification)
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **낮음~중간** |
| 데이터 확보 | 자체 이벤트 기반 (부킹 상태 변경, 선박 일정 변경 등) |
| 카고플랜 적합성 | **높음** — 고객 커뮤니케이션 자동화 |
| 핵심 기능 | 알림 목록, 검색, 실시간 푸시 |
| 자체 구축 이점 | 이메일/카카오톡 등 멀티채널 확장 가능 |

### ⚠️ Tier 2: 조건부 추천 (데이터 확보가 관건)

#### 5. 해상 운임 조회 (운임 추이)
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **높음** |
| 데이터 확보 | **핵심 난관** — 선사 실시간 운임 데이터 확보 어려움 |
| 카고플랜 적합성 | **높음** — 화주 대상 영업 도구로 가치 있음 |
| 핵심 기능 | 컨테이너 유형별 운임 차트, POL/POD 검색 |
| 자체 구축 조건 | 최소 주요 거래 선사와의 데이터 공유 협약 필요 |

**주의**: 서프는 300+ 파트너 포워더 + 현대글로비스 배경으로 데이터 확보. 카고플랜이 동일 수준 달성은 어려움. **자사 거래 운임 데이터 기반으로 제한적 제공**은 가능.

#### 6. 컨테이너선 운임지수 (SCFI, KCCI, CCFI)
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **낮음~중간** |
| 데이터 확보 | **공개 데이터** — 상해항운교역소(SCFI), 한국해양수산개발원(KCCI) 등 |
| 카고플랜 적합성 | **중간** — 참고 정보 제공용 |
| 핵심 기능 | 기간별 필터(1개월~5년), 차트, PDF 다운로드 |
| 자체 구축 이점 | 공개 데이터이므로 크롤링/API로 수집 가능 |

#### 7. 수출입 물동량 조회
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **중간** |
| 데이터 확보 | **관세청 공공데이터** 활용 가능 |
| 카고플랜 적합성 | **중간** — 영업 참고 자료 |
| 핵심 기능 | 기간/국가별 물동량 테이블 |
| 자체 구축 조건 | 관세청 API 연동 + 데이터 시각화 |

#### 8. 해운 데이터 맵
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **높음** |
| 데이터 확보 | 해양수산부/세관 데이터 + 선사 주가(증권 API) |
| 카고플랜 적합성 | **중간** — 대시보드로서 가치 있으나 핵심 업무와 거리 |
| 핵심 기능 | Port별 물동량, 선사 주가 추이, 인기 뉴스 |
| 자체 구축 조건 | 다양한 외부 데이터 소스 통합 필요 |

#### 9. 블로그 / 물류 콘텐츠
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **낮음** |
| 데이터 확보 | 자체 콘텐츠 생산 필요 |
| 카고플랜 적합성 | **중간** — SEO/마케팅 채널, 전문성 어필 |
| 핵심 기능 | 카테고리별 아티클, 검색, 뉴스레터 구독 |
| 자체 구축 조건 | **콘텐츠 생산 역량** 확보 필수 (주 1~2회 발행) |

### ❌ Tier 3: 자체 구축 비추천 (비효율적)

#### 10. 마켓 플레이스 (운임 비교 + 부킹)
| 항목 | 평가 |
|------|------|
| 구축 난이도 | **매우 높음** |
| 데이터 확보 | 다수 선사/포워더 운임 데이터 확보 불가능 |
| 카고플랜 적합성 | **부적합** — 포워더가 마켓플레이스를 운영하는 것은 이해충돌 |
| 비추천 사유 | 카고플랜은 **포워더**이므로, 다른 포워더의 운임을 비교하는 마켓플레이스 운영은 비즈니스 모델과 상충. 서프/트레드링스 같은 **플랫폼 사업자**의 영역. |

#### 11. 맞춤 견적 의뢰 플랫폼
| 항목 | 평가 |
|------|------|
| 비추천 사유 | 마켓플레이스와 동일한 이유. 다만 **자사 고객 전용 견적 요청 시스템**으로 변형 가능 (→ Tier 1의 Booking 관리에 통합) |

#### 12. 프리미엄 데이터 판매
| 항목 | 평가 |
|------|------|
| 비추천 사유 | 데이터 수집/가공/판매는 별도 사업. 카고플랜의 핵심 역량(포워딩)과 거리가 멀고, 서프/트레드링스 대비 데이터 경쟁력 확보 어려움. |

## 최종 결론

카고플랜이 서프와 계약 없이 자체 구축하기에 **가장 유효한 기능 4가지**:

1. **카고 트래킹** — 고객 가치 최고, 선사 API 활용으로 구현 가능
2. **Booking 관리 시스템** — 내부 운영 + 고객 서비스 동시 해결
3. **실시간 알림** — 위 두 기능의 가치를 극대화하는 보조 기능
4. **선박 입출항 알림** — 공공데이터 기반, 실무 즉시 활용

반면, **마켓플레이스/운임비교/데이터 판매**는 플랫폼 사업자(서프, 트레드링스)의 영역으로, 포워더인 카고플랜이 직접 구축하는 것은 비효율적이고 비즈니스 모델과 상충됨.

> **핵심 메시지**: "남의 기능을 전부 따라 만들 필요 없다. 포워더로서 고객에게 직접 가치를 주는 기능에 집중하라."
